### 合并写入  
由于字段过多，这样操作比较方便。  
  
每次写入时，不需要设置的值删掉,或使用null代替。  
  
更新使用同一条语句，不需要更新的值删掉,或使用null代替。  
  
```  
insert into t (  
  key,  
  字段1,  
  字段2,  
  ...  
)  
values (    
  1,     -- key  
  null,  -- 字段1  
  null,  -- 字段2  
  ...  
  val,   -- 字段n  
  ...  
)  
on conflict (id)  
do update  
set   
  xx = coalesce(excluded.xx, t.xx),   -- 字段1   
  ...  
  -- 写全，这一段固定，不需要修改。  
;  
```
