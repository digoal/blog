## 微软宣布推出 Azure HorizonDB  
  
**作者:** Affan Dar，微软 PostgreSQL 工程副总裁；Charles Feddersen，微软 PostgreSQL 项目管理合伙人总监  
  
今天在 **Microsoft Ignite** 大会上，我们很高兴地宣布 **Azure HorizonDB** 的**预览版**（preview）正式推出，它是一种**完全托管**（fully managed）的 **Postgres兼容的数据库服务**（Postgres-compatible database service），旨在满足现代企业工作负载的需求。**Azure HorizonDB** 的**云原生架构**（cloud native architecture）提供了**高可扩展共享存储**（highly scalable shared storage）、**弹性横向扩展计算**（elastic scale-out compute）以及针对运行任何规模云应用程序而优化的**分层缓存**（tiered cache）。  
  
**Postgres** 正在全球范围内推动行业转型，并以前所未有的速度成为各行业现代数据解决方案的基础。对于开发者而言，凭借其丰富的扩展集、开源 **API** 以及庞大的工具和库生态系统，它是构建新应用的**首选数据库**（database of choice）。与此同时，在全球工作负载的另一端，企业也越来越多地转向使用 **Postgres** 来现代化其现有应用。  
  
**Azure HorizonDB** 旨在支持整个**工作负载范围**（workload spectrum）内的应用程序，从新应用的第一行代码到大规模、**任务关键型解决方案**（mission-critical solutions）的迁移。开发者可受益于强大的 **Postgres** 生态系统和与 **Azure** 高级 **AI** 功能的无缝集成，而企业则能获得安全、高可用且高性能的云数据库来托管其业务应用程序。无论是从零开始构建还是改造遗留基础设施，**Azure HorizonDB** 都能让您自信地进行创新和扩展，无论是当下还是未来。  
  
**Azure HorizonDB** 为 **PostgreSQL** 带来了新的性能和可扩展性水平。其**横向扩展计算架构**（scale-out compute architecture）支持主节点和**副本节点**（replica nodes）上多达 3,072 个 **vCores**（虚拟核心），而**自动扩展共享存储**（auto-scaling shared storage）支持高达 **128TB** 的数据库，同时提供**亚毫秒级多可用区提交延迟**（sub-millisecond multi-zone commit latencies）。这种存储创新使 **Azure HorizonDB** 在处理**事务性工作负载**（transactional workloads）时，吞吐量比开源 **Postgres** 高出 **3 倍**。  
  
**Azure HorizonDB** 从第一天起就具备企业级能力。凭借对 **Entra ID**、**私有端点**（Private Endpoints）和**数据加密**（data encryption）的原生支持，它为存储在云中的敏感数据提供了合规性和安全性。默认情况下，所有数据都会在**可用区**（availability zones）之间进行复制，维护操作是透明的，可实现**近零停机时间**（near-zero downtime）。备份是完全自动化的，与 **Azure Defender for Cloud** 的集成提供了针对高度敏感数据的额外保护。总而言之，**Azure HorizonDB** 提供了企业级的安全性、合规性和可靠性，使其可立即用于业务用途。  
  
自 **ChatGPT** 推出以来，新的 **AI** 应用程序呈爆炸式增长，而 **Postgres** 之所以成为首选数据库，很大程度上归功于其**向量索引支持**（vector index support）。**Azure HorizonDB** 通过两项关键功能进一步扩展了 **Postgres** 的 **AI** 能力。我们正在为 **DiskANN 向量索引**（DiskANN vector index）引入高级过滤功能，该功能能够将**查询谓词下推**（query predicate pushdowns）直接推送到**向量相似性搜索**（vector similarity search）中。这与 **pgvector HNSW** 相比，在保持准确性的同时，显著提升了性能和可扩展性，非常适合在 **Postgres** 中的事务性数据上进行相似性搜索。第二项功能是内置的 **AI 模型管理**（AI model management），可无缝集成来自 **Microsoft Foundry** 的**生成式、嵌入式和重新排序模型**（generative, embedding, and reranking models），供开发者在数据库中零配置使用。  
  
除了增强的向量索引和简化的模型管理来构建强大的新 **AI** 应用之外，我们还很高兴地宣布 **Microsoft 的 VS Code PostgreSQL 扩展**（Microsoft's PostgreSQL Extension for VS Code）正式发布，它为 **Postgres** 开发者提供了最大限度提高生产力的工具。使用此扩展，**GitHub Copilot** 可**上下文感知** （context aware） **Postgres** 数据库，这意味着更少的提示和更高质量的答案。在 **Ignite** 版本中，我们添加了实时监控和一键式 **GitHub Copilot 调试**（GitHub Copilot debugging）功能，其中**代理模式**（Agent mode）可以直接从性能监控仪表板启动，以诊断 **Postgres 性能问题**（Postgres performance issues）并指导用户进行修复。  
  
**Alpha Life Sciences** 是一家现有的 **Azure** 客户：  
  
“我对 **Azure HorizonDB** 如何赋能我们的 **AI** 开发感到非常兴奋。它对 **Vector DB**、**RAG** 和**智能体 AI**（Agentic AI）的无缝支持，使我们能够直接在可靠的 **Postgres** 基础上构建智能功能。有了 **Azure HorizonDB**，我可以专注于推进 **AI** 能力，而不是管理基础设施的复杂性。这是一个智能、前瞻性的解决方案，完美契合了我们设计和交付 **AI** 驱动型应用程序的方式。”—— Pengcheng Xu，**Alpha Life Sciences** 首席技术官（CTO）  
  
对于正在将应用程序现代化到云中 **Postgres** 的企业来说，**Azure HorizonDB** 的安全性和可用性使其成为理想的平台。然而，对于大型**遗留代码库转换**（legacy codebase conversions）而言，这些迁移通常是复杂且耗时的。为了简化这一点并降低风险，我们很高兴地宣布 **GitHub Copilot 驱动的 Oracle 迁移**（GitHub Copilot powered Oracle migration）预览版已内置到 **VS Code** 的 **PostgreSQL** 扩展中。工程团队可以在 **VS Code** 中使用 **GitHub Copilot**，通过富代码编辑、版本控制、文本创作和集成开发环境中的部署，自动化复杂数据库代码的端到端转换。  
  
**Azure HorizonDB** 是下一代**完全托管**（fully managed）、**云原生**（cloud native）的 **PostgreSQL** 数据库服务。它构建在最新的 **Azure** 基础设施和**最先进的云架构**（state-of-the-art cloud architecture）之上，已为最苛刻的应用程序工作负载做好准备。  
  
除了我们在 **Azure** 中的托管 **Postgres** 服务组合外，微软还深度投资于**开源 Postgres 项目**（open source Postgres project），是 **PostgreSQL** 项目的顶级**企业上游贡献者**（corporate upstream contributors）和赞助商之一，微软雇佣了 19 位 **Postgres** 项目贡献者。作为一家**超大规模 Postgres 供应商**（hyperscale Postgres vendor），积极参与开源项目至关重要。它使我们能够更好地在 **Azure** 中**深入到硬件层面**（down to the metal）支持客户，并将我们运行大规模 **Postgres** 的经验教训回馈给社区。我们致力于继续投入，推动 **Postgres** 项目向前发展，团队已积极为将于 2026 年发布的 **Postgres 19** 做出贡献。
