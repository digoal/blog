# ZeroClaw 简介：超轻量、全能的 AI 代理操作系统  
  
ZeroClaw 是一个用 Rust 编写的 AI 代理基础设施——你可以把它理解成一个专门运行 AI 代理的“操作系统”。它让你能**在任何硬件上部署、任意替换组件**，同时保持极致的性能和安全性。  
  
## 为什么需要 ZeroClaw？  
  
现有的 AI 代理框架（比如 OpenClaw）往往很重：内存占用几百 MB、启动慢、依赖复杂。ZeroClaw 的目标是**把代理跑在 10 美元的硬件上，内存占用不到 5MB**——比 OpenClaw 省 99% 的内存，比 Mac mini 便宜 98%。  
  
- 🦀 **100% Rust 编写**：编译成单个二进制文件，启动毫秒级，内存极小。  
- 🔌 **组件全可插拔**：AI 模型、聊天渠道、记忆存储、工具、安全策略……所有模块都可以通过配置文件替换，不改代码。  
- 🔒 **安全默认**：网关默认只绑本地，必须配对才能使用，文件系统默认沙盒化，支持频道白名单。  
- 🌍 **随处运行**：ARM、x86、RISC-V 都支持，甚至可以在树莓派、旧手机上跑。

## 快速开始（3 分钟上手）  
  
### 1. 安装  
  
#### macOS / Linux（推荐）  
```bash  
# 一键安装（自动装 Rust + 编译）  
curl -fsSL https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/main/scripts/bootstrap.sh | bash  
```  
或者用 Homebrew：  
```bash  
brew install zeroclaw  
```  
  
#### Windows  
- 先安装 Visual Studio Build Tools（选“C++桌面开发”工作负载）  
- 再用 winget 安装 Rust：`winget install Rustlang.Rustup`  
- 最后从 [Release 页面](https://github.com/zeroclaw-labs/zeroclaw/releases) 下载 exe，或 `cargo install zeroclaw`  
  
### 2. 配置 AI 提供商  
  
运行向导，填入你的 API Key（比如 OpenRouter、OpenAI 等）：  
```bash  
zeroclaw onboard --api-key sk-... --provider openrouter  
```  
这会在 `~/.zeroclaw/config.toml` 生成配置文件。  
  
### 3. 开始聊天  
  
```bash  
zeroclaw agent -m "你好，ZeroClaw！"  
```  
或者进入交互模式：  
```bash  
zeroclaw agent  
```  
  
### 4. 通过 Telegram 使用（可选）  
  
1. 在 Telegram 搜索 @BotFather 创建机器人，拿到 token。  
2. 编辑配置 `~/.zeroclaw/config.toml`，加入：  
   ```toml  
   [channels_config.telegram]  
   bot_token = "你的:token"  
   allowed_users = ["你的用户名"]   # 不加 @，也可用数字 ID  
   ```  
3. 启动后台守护进程：  
   ```bash  
   zeroclaw daemon  
   ```  
现在你的 Telegram 机器人就能和 ZeroClaw 对话了。  
  
## 配置文件极简示例  
  
```toml  
api_key = "sk-..."                # 你的 API Key  
default_provider = "openrouter"   # 默认模型提供商  
default_model = "anthropic/claude-sonnet-4-6"  
  
[memory]  
backend = "sqlite"                 # 记忆存在本地 SQLite  
auto_save = true  
  
[gateway]  
port = 42617                       # 本地网关端口  
require_pairing = true             # 需要配对码才能连接  
  
[autonomy]  
level = "supervised"               # 代理自主级别：只读/监督/全自动  
workspace_only = true               # 限制只能访问工作目录  
```  
  
## 常用命令速览  
  
| 命令 | 作用 |  
|------|------|  
| `zeroclaw agent` | 交互式聊天 |  
| `zeroclaw daemon` | 启动后台（接收 Telegram 等渠道消息） |

## 架构亮点：一切皆 Trait  
  
ZeroClaw 的核心是 Rust 的 **trait（接口）** 。每个子系统都是一个 trait，你可以通过配置文件替换实现，完全不用改代码：  
  
- **模型**：`Provider` trait – 支持 OpenAI 兼容、Anthropic、Ollama 等  
- **渠道**：`Channel` trait – Telegram、Discord、邮件……  
- **记忆**：`Memory` trait – SQLite、PostgreSQL、甚至纯文本文件  
- **工具**：`Tool` trait – shell、文件、浏览器、硬件控制……  
- **安全**：`SecurityPolicy` trait – 白名单、沙盒、速率限制  
- **隧道**：`Tunnel` trait – Cloudflare、Tailscale、ngrok 等  
  
这意味着你可以：  
- 把聊天记录存到自己的 PostgreSQL 数据库  
- 让代理通过 WhatsApp 接收消息  
- 添加自定义工具（比如控制智能家居）  
- 全部只改配置，不动代码。  
  
## 安全设计（开箱即用）  
  
- **网关默认本地**：`127.0.0.1`，绝不暴露公网，除非你配了隧道。  
- **强制配对**：第一次启动生成 6 位配对码，客户端必须发配对请求才能拿 token。  
- **文件系统沙盒**：默认只能访问工作目录，禁止读写系统敏感路径（如 `/etc`、`~/.ssh`）。  
- **渠道白名单**：每个频道（如 Telegram）默认拒绝所有人，必须手动添加允许的用户 ID。  
- **加密存储**：API Key 等敏感信息用本地密钥加密存放。
