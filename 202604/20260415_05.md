解决 claude code 使用第三方模型时, Web Search 无法调用的问题, 方法类似.   
  
```
⏺ Web Search("杭州 天气")
  ⎿ Did 0 searches in 209ms

  Thought for 3s (ctrl+o to expand)

⏺ WebSearch 接口仍不可用，改用 WebFetch 拉取天气网站。

⏺ Fetch(https://www.weather.com.cn/weather/101210101.shtml)
  ⎿ Received 89.7KB (200 OK)

⏺ Fetch(https://weather.cma.cn/web/weather/58457.html)
  ⎿ Received 80.4KB (200 OK)
```
  
错误原因: 官方内置的 web_search 依赖于 Anthropic API 的新协议特性（tool_reference 模块），而大部分第三方网关尚未适配这些协议，导致搜索功能静默失效.
  
使用 minimax 时, 可使用它的 mcp 功能.
```
claude mcp add -s user MiniMax --env MINIMAX_API_KEY=你的API密钥 --env MINIMAX_API_HOST=https://api.minimaxi.com -- uvx minimax-coding-plan-mcp -y
```
  
在终端里输入 claude 重启 Claude Code。
  
进入 Claude Code 的交互界面后，输入 `/mcp` 命令查看已加载的工具列表。
  
如果配置成功，你应该能在列表中看到 `web_search` 和 `understand_image` 这两个工具。
  
此时还不够, 还需要编辑 `~/.claude/CLAUDE.md` ，加入以下规则：
```
# WebSearch 重定向规则

## 核心规则
当需要联网搜索时，**禁止使用内置的 `web_search`**，必须使用 MiniMax MCP 提供的搜索工具。

## 工具调用规范
- 执行搜索时，必须调用 `mcp__MiniMax__web_search`
- 如果该工具不可用，告知用户后停止，不要尝试任何其他搜索方式

## 示例
用户问："搜索一下今天的新闻"
正确做法：调用 `mcp__MiniMax__web_search(query="今天的新闻")`
```
  
然后重启 Claude Code 再测试, 就会使用 minimax 的 mcp__MiniMax__web_search 了.

但是我有个问题, 如果当前项目 以及 HOME 目录的 `.codex` 和 `.agent` 目录中同时存在同名 `skill` / `AGENTS.md` , 优先级是什么样的?   
从高到低（覆盖顺序）, 顺序其实符合所有现代开发工具的一个原则：“就近原则 + 项目优先 + 用户兜底”   
1️⃣ `当前项目目录 /.codex`  
2️⃣ `当前项目目录 /.agents`  
3️⃣ `用户 HOME ~/.codex`  
4️⃣ `用户 HOME ~/.agents`

# 分析 commit log 和代码 , 生成文章: PostgreSQL 19 最值得期待的特性  
# 提示词如下  
1、首先, 详细分析该项目所有 commit log (仅需分析简短描述, 不要分析完整描述, 不要分析代码的 diff 内容), 剔除被回退的 commit, 整理出对 DBA(数据库管理员)和用户(应用开发者) 最有价值的特性和改进以及其相关的 commit short ids;  
  
2、对 commit id 进行分类, 解决同样问题的 commit ids 归到一组;  
  
3、然后再详细分析这些 commit ids 的完整描述 (不需要分析代码的 diff 内容, 除非有必要进行更深的理解或拿到实践示例);
