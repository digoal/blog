## 背景  
claude、claude web、codex 都自带了 Skill Creator 技能(例如 `.codex/skills/.system/skill-creator`), 可将你自己的技能编写成 SKILL 供日常调用, 也可以让 AI 自动分析某个岗位的能力模型(哪里找? 招聘网站一大把, 随便搜一个招聘需求都可以看到任职要求、岗位职责等介绍.), 编写各个相关的 SKILL.  
  
下面是我最近写的一些技能, 详见: https://github.com/digoal/blog/tree/master/skills  
  
今天就给大家来介绍一下, 如何创建 SKILL.  
  
## skill 介绍  
- `daily-finance`：每日联网采集并校验高质量财经新闻、市场数据和关键事件，生成可直接发布的公众号财经日报，并作为后续深度分析的事实底稿。  
  
- `finance-core-analysis`：基于 `daily-finance` 的事实底稿，再联网复核关键数据，用流动性、利率、风险偏好、资金流、政策和资产负债表模型生成可发布的深度财经分析。  
  
- `finance-explosive-article`：基于前两份财经文档和最新外部数据校验，用“第一性原理 + 反直觉 + 系统模型”的德哥风格生成公众号爆款财经文章。  
  
- `digoal`：基于 digoal/德哥博客沉淀，面向 PostgreSQL、AI+数据库、开源生态、技术文章和架构判断，输出证据驱动、场景优先、可验证的德哥式分析与方案。  
  
- `github-weekly-trending`: 输入从 `https://github.com/trending?since=weekly&spoken_language_code=` 拷贝的内容, 编写本周热门开源项目文章, 输出到当前项目 markdown 目录中.  
  
- `open-source-project-article`: 输入开源项目地址, 深度分析该开源项目, 输出到当前项目 markdown 目录中.  
  
- `marketing-wechat-operator`: `微信公众号运营`, 编写爆款文章.  
  
- `postgres-commit-history-article`: 先进入 postgres 项目目录, 输入 `commitid1 commitid2` , 分析并解读这两个 commitid 中间的所有提交 (也包括这两个 commit), 输出到当前项目 markdown 目录中.  
  
- `paper-interpretation`: 输入论文 PDF 或论文 URL , 通俗易懂解读论文. 例如用于解读 AI 论文 https://huggingface.co/papers/trending https://arxiv.org/abs/2604.14141 https://arxiv.org/abs/2508.02739  
  
- `pgfaq`: clone https://github.com/postgres/postgres 源码, 将其作为项目目录. 输入 PostgreSQL 相关的问题, 将回答结果保存到项目目录的 markdown 子目录中. 回答时会参考代码、文档和deepwiki, 并对回答内容正确性进行校验.

## 编写方法  
给一些例子:

````  
编写一个skill: pgfaq , 用于解答任何 postgresql 数据库问题. 将解答结果以 markdown 格式保存到当前项目的 markdown 目录中.  
  
解答方式:  
1、首先确保正在 postgres 源码目录中, 根据问题搜索必要的相关的文档或代码;

最后校验解答内容的正确性,  
- 将内容发送给 deepwiki mcp 进行确认, 并通过相关代码、文档确认内容正确性. 并根据建议进行修正, 重复校验和修正步骤, 直到解答结果正确.  
  
将解答结果以 markdown 格式保存到当前项目的 markdown 目录中.  
  
如果问题涉及 postgresql 开源项目以外的项目, 例如 PG 生态周边的插件, 工具等, 请直接拒绝回答.

2、  
````  
将下面解读论文的方法整理成 skill, 输入论文的 PDF 文件 或论文 PDF URL, 输出 markdown 格式的文件, 保存到当前项目的 markdown 目录中.  
  
要求在消化完论文的全部内容(也包括论文中的表格、图片等信息)后, 通俗易懂的解读这篇论文, 既要体现出论文包括工业价值和学术价值在内的核心价值, 又要让更多人能看懂, 还要图文并茂(关键时刻使用 text,mermaid,svg 等 markdown 支持的图格式).  
  
解读方法和步骤如下:  
  
### 第1步：定位论文 — 先给读者一个"地图"  
  
先告诉读者"这是什么、跟我有什么关系、值不值得读"。提升读者后续的阅读动力。  
- 用1-3句话说清楚这篇论文解决了什么现实问题  
- 标注它的学术价值（在学界填补了什么空白）和工业价值（在业界能用来做什么）  
- 给出一个直觉性的类比："这篇论文就像是给AI配了一个完整的投资团队"  
  
### 第2步：知识地图 — 前置知识体系化  
  
把精读论文之前的前置知识画成一棵树，分层标注难度和重要性  
  
```  
核心概念（必须懂）→ 支撑概念（有助于理解）→ 扩展概念（感兴趣再看）  
```  
  
用图表、类比、举例三件套来讲每个概念，而不是只给定义。  
  
### 第3步：论文精读 — 按"问题-方案-验证"三段式解读  
  
用记者的5W1H框架拆解论文  
  
| 问题 | 对应论文结构 |  
|------|------------|  
| Why：为什么要做这个研究？ | Introduction / Motivation |  
| What：提出了什么方法/系统？ | Method / Architecture |  
| How：具体怎么实现的？ | Technical Details |  
| So What：结果怎么样？ | Experiments / Results |  
| Now What：对我们意味着什么？ | 工业/学术价值总结 |  
  
关键内容用对比来讲（和已有方法比，和直觉理解比），比单纯讲原理更易懂。  
  
### 第4步：术语词典 — 术语要讲"为什么这样命名"  
  
每个术语用"是什么 + 为什么重要 + 现实类比"三句话模板讲解  
  
例如：  
  
Multi-Agent System（多智能体系统）  
- 是什么：多个AI角色分工协作完成任务  
- 为什么重要：单个AI容易有盲区，分工后互相校验更可靠  
- 现实类比：就像一家公司有分析师、交易员、风控官，各司其职  
  
### 第5步：批判性评估 — 让读者学会独立思考  
  
真正理解一篇论文，不只是"看懂它说什么"，还要能评价它的局限性。这是学术素养的体现，也是工业落地时最关键的判断。  
- 论文的假设前提是否在现实中成立？  
- 实验设计有没有可质疑之处？  
- 这个方法在什么场景下会失效？  
- 未来可以怎么改进？  
````

下面我再提供一份读特斯拉财报时提出的一系列连环问题: 

这家公司本年度最核心的收入来源是什么？
与去年相比，毛利率发生了什么变化？原因是什么？
财报中提到的‘主要经营风险’有哪些？
经营活动产生的现金流是否能够覆盖其净利润？
从投资角度总结财报
财报中是否存在这些坑:
````
### 1. 虚假的“纸面富贵”：利润好，但钱回不来 
**坑点：** 营收和利润连年增长，股价看似便宜，但公司其实一直在倒贴。 
* **识别指标：** **应收账款（AR）** 增速远超 **营业收入** 增速。 
* **真相：** 公司可能为了冲业绩，强行把货塞给经销商（即便对方没给钱）。这叫“提前确认收入”。 
* **避坑指南：** 关注 **“经营活动产生的现金流量净额”** 。如果利润是正的，现金流常年是负的，这个利润就是“白纸一张”。 
### 2. 垃圾堆里的“虚资产”：存货和应收账款爆棚 
**坑点：** 资产负债表看起来家底很厚，但全是“垃圾”。 
* **识别指标：** **存货周转率** 持续下降。 
* **真相：** 
  * **存货坑：** 仓库里的货可能已经过时卖不掉了（比如去年的新款手机），但公司不计提减值，假装它们还值钱。 
  * **应收账款坑：** 那些欠款可能已经变成了坏账（对方公司都倒闭了），公司却一直挂在账上。 
* **避坑指南：** 翻看附注中的 **“账龄分析”** 。如果 1 年以上的欠款占比越来越高，这就是个“大坑”。 
### 3. “财务黑盒”：其他应收款与预付款项 
**坑点：** 账面上有很多莫名其妙的借款。 
* **识别指标：** **其他应收款** 占总资产比例异常（如 >5%）。 
* **真相：** 就像我们之前讨论的，这通常是大股东 **“掏空”** 公司的手段。公司把钱以“借款”或“预付货款”的名义转给关联方，实际上是有去无回。 
* **避坑指南：** 看到这两个科目特别大，直接视为风险信号，除非附注里有极其合理的商业解释。 
### 4. “商誉雷区”：高价买来的“幻象” 
**坑点：** 为了扩张疯狂收购，账面上挂着巨额商誉。 
* **识别指标：** **商誉 / 净资产 > 30%** 。 
* **真相：** 经济好时相安无事，一旦被收购公司业绩滑坡，商誉就会发生 **“大减值”** ，一年亏掉过去十年的利润。 
* **避坑指南：** 检查被收购公司是否完成了 **“业绩承诺”** 。如果踩线完成或没完成，明年大概率要暴雷。 
### 5. “大存大贷”怪象：家里有矿，还要借高利贷 
**坑点：** 账面上现金极多（如 50 亿），但同时背负巨额高息贷款（如 40 亿）。 
* **识别指标：** **货币资金** 与 **短期借款** 同时高企。 
* **真相：** 逻辑不通。如果你有钱，为什么要付利息去借钱？ 
  * 真相 A：账上的钱是假的（虚增出来的）。 
  * 真相 B：钱被大股东质押或冻结了，公司根本动不了。 
* **避坑指南：** 这种公司往往是财务造假的“重灾区”（参考康美药业）。

* **逻辑：** 研发投入本来应该是费用（直接扣减利润），但公司可以主张这个研发项目“很有前景”，将其计入 **“开发支出”** （资产表），等研发成功后再慢慢摊销。

这是很多大集团最隐蔽的手段。 
* **逻辑：** 按照规则，如果持股比例低于 50%（且没有实际控制权），这家公司的债务和亏损就不需要合并到上市公司的报表里，只需按比例记录投资损益。

* **逻辑：** 当公司今年注定要亏损时，管理层干脆“一亏到底”，把未来几年的潜在亏损（如商誉、坏账、存货）全部在今年一次性计提减值。 
* **隐蔽点：** 这种行为在当年看起来很惨，但实际上是 **“利好出尽”** 。因为把资产减值到底了，明年的折旧少了，包袱轻了，稍微做点生意利润就能“爆发式增长”。 
* **识别：** 关注亏损当年的减值是否合理。如果减值额高得离谱，且管理层刚换届，这通常是新官上任在“洗大澡”，为明年的业绩爆发铺路。

为一个企业提供真正有价值的规划，其核心在于诊断“现状”与“未来可能性”之间的差距。规划不是凭空创造，而是搭建一座从A点到B点的可行桥梁。

这属于企业诊断的范畴，要看三张地图：

    -   客户资产：客户群体画像、集中度（是否依赖大客户？）、复购率、净推荐值（NPS）。客户为什么付费？解决了什么痛点？
    -   运营效率：人效（人均产出/利润）、库存周转率、交付周期、研发转化率等关键运营指标。

-   组织与人（软实力）
    -   核心团队：创始人背景、核心高管的履历、决策机制风格。是一言堂还是集体决策？核心团队的优势与盲区是什么？
    -   人才结构：关键岗位的板凳深度、员工流失率、人才梯队是否断层。普通员工对战略是否有感知？
    -   文化与价值观：墙上挂的是标语，还是大家真正相信并实践的行为准则？遇到冲突时，什么原则是最后防线？

-   资源与能力根基
    -   核心资源：独家的牌照、专利、矿产、品牌资产、供应链关系、数据资产。
    -   核心能力：是营销驱动、技术驱动还是运营效率驱动？这种能力是否容易被模仿？要把核心能力拆解成流程，看它是否稳定地附着在系统上，而不是依赖个别能人。

#### 针对未来：看清“想去哪，能去哪”

这部分是战略意图与进化能力的评估：

-   战略意图与野心：老板做企业的终极目的是什么？是做成百年老店、上市套现，还是待价而沽？这直接决定了规划的时间维度和激进程度。
-   创新与进化能力：过去面对重大市场变化时，企业的反应速度和结果如何？以三年为尺度，现在的企业与三年前相比，最大的结构性变化是什么？还是只在做同质化的规模扩张？
-   可迁移的资产：现有的技术、能力、客户关系，能平移到哪些新领域？这是多元化规划的基础。

### 二、必须了解的“外部信息”

这部分是你的“地图和天气预报”，目的是判断企业所处的环境与机会窗口。

#### 针对当下：看清“战场状况”

关键是结构分析，而非现象罗列：

-   市场结构地图
    -   市场规模与利润池：整个行业的规模、增速，以及利润主要集中在哪个环节？你在高利润区还是搬箱区？
    -   价值链与生态位：上下游是怎么分工的？谁在定价上最有话语权？你的上下游在发生什么变化？（你的天花板和压力源都在这里）

-   客户与需求洞察
    -   购买决策流程：客户是如何获知、比较、购买、使用和复购的？决策者、购买者、使用者是否分离？客户未被言明的、忍耐已久的最深层痛点是什么？

-   竞争与合作图谱
    -   直接对手：同一客群、同一需求，为何客户选他不选你？把他的产品买来自己用一遍。
    -   替代品：解决问题的全新方式，哪怕来自其他行业（如高铁替代短途航空）。
    -   潜在进入者：巨头们（尤其是科技巨头）有没有看上这块蛋糕？为什么还没动手？

-   宏观与政策即时温度：当下的监管风向、信贷环境、就业市场等，这些直接影响企业当下的成本与订单。

#### 针对未来：看清“风向与洋流”

-   技术的非连续性演变：不必追逐每一波热点，但必须警惕有没有一种技术，能让你的核心业务成本结构发生数量级变化，或完全颠覆现有的解决方案？ 这是最底层的威胁或机遇。
-   社会与人口的结构性变迁：人口老龄化、新一代（如Z世代）的价值观变迁、家庭结构小型化、城镇化下半场等慢变量，是未来十年最大的确定性，会重塑所有行业的底层需求逻辑。
-   经济与政策的长期锚点：国家的长期规划（如碳中和、数字经济）、全球产业链重构（从效率优先到安全优先）的趋势，决定了未来10年的基础经营环境。
-   价值观驱动的消费迁移：从“拥有”到“使用”，从“功能”到“意义”，本土文化自信等价值观变化，会催生全新的品类和品牌机会。

### 最后，提出有价值的规划，关键在于建立联系

当你在内部信息中看到企业的“局限性”，在外部信息中看到“可能性”时，规划其实就是在回答以下问题：

-   当下对当下：用外部市场的高利润区，对比内部业务的低效区，找出现有业务的效率提升和利润迁移机会。

### 第一部分：缺少的关键数据

新闻告诉你“发生了什么”以及市场“怎么想”，但以下数据能告诉你“市场已经定价了多少”以及“未来可能发生什么”。

新闻是催化剂，但价格和成交量数据能验证市场的真实态度。
-   量价关系数据：当利好新闻出现时，个股/行业是放量上涨（真突破）还是缩量上涨（可能诱多）？当利空新闻出现时，是放量下跌（恐慌抛售）还是缩量下跌（惜售，可能见底）？没有这些数据，你无法判断新闻的有效性。
-   资金流向数据：
    -   主力资金净流入/流出：新闻背后，大资金是在借利好出货，还是在借利空吸筹？
    -   北向资金（A股）/ 美股ETF资金流：聪明钱的流向是重要的多空信号。
-   技术指标与关键点位：
    -   支撑/阻力位、均线系统：新闻的冲击，是刚好打到强阻力位回落，还是放量突破了关键年线？技术分析提供了具体的战场地图。

#### 2. 市场结构与情绪数据（衡量市场的“温度”）
新闻分析多是主观的，而结构与情绪数据提供了客观的量化尺规。
-   市场宽度数据：
    -   涨跌家数比：指数在涨，是少数龙头拉动的（外强中干），还是多数股票普涨（健康上涨）？

    -   期权看跌/看涨比率：衡量市场参与者的对冲和投机行为，是极佳的反向指标。

### 第二部分：需要的预测方法论

你需要将上述数据和新闻进行系统化分析，构建一个可重复的框架。

#### 1. 建立“新闻-数据”的映射与验证框架
不要只根据新闻本身判断多空，而是建立一套“信号-验证”机制。

-   案例：
    -   新闻：某行业出台重大利好政策，财经媒体一致看多。
    -   验证1（市场行为）：查看该行业ETF是否在新闻后放量突破关键阻力位，且主力资金大幅净流入。若缩量上涨，高度警惕。
    -   验证2（情绪与结构）：该行业新高个股数量是否在同步增加？若只有几只龙头在涨，力度有限。
    -   验证3（预期与拥挤度）：查看机构在此新闻前对该行业的配置比例。如果已严重超配（拥挤），则此新闻可能是“靴子落地”的出货信号。

#### 2. 构建一个简单的三因子定价模型
未来一周的股价变化 ≈ 盈利预期的变化 + 估值（情绪）的变化 + 流动性溢价的变化。

-   盈利预期因子：将新闻分类。哪些直接影响企业未来1-2周的盈利预期？（如超预期的财报、行业数据、重大订单）。通过跟踪分析师对个股/行业的盈利预测修正数据来量化。
-   估值/情绪因子：新闻如何改变风险偏好？通过VIX、新高新低比、资金流来量化市场愿意为该行业支付多少溢价。
-   流动性因子：新闻是否改变了流动性预期？（如央行逆回购操作、美联储官员讲话）。通过Shibor、国债收益率、北向资金来量化。

预测时，给这三个因子分配权重：
-   在财报季，盈利预期因子权重最高。
-   在宏观政策出现重大转向时，流动性因子是决定性力量。
-   在横盘震荡、无主线的市场，估值/情绪因子（资金博弈行为）主导短期行情。

#### 3. 运用“预期差”作为核心决策依据
这是盈利的核心：“比新闻更重要的是，市场已为新闻定价了多少。”

-   创建“预期-现实”矩阵：
    -   好消息 + 高预期（已充分定价）：= 利好出尽是利空（看跌）。
    -   好消息 + 低预期（未定价）：= 催化剂（看涨）。
    -   坏消息 + 高预期（未料到）：= 黑天鹅（大跌）。
    -   坏消息 + 低预期（已充分消化）：= 利空出尽是利好（看涨）。
-   运用方法：拿到一条新闻，你的第一反应不是“这是好消息”，而是“这个好消息市场之前想到了吗？” 通过对比新闻内容和当前股价位置、分析师预测、估值水平、持仓拥挤度，来判断这个“预期差”的大小和方向。

#### 4. 采用“自上而下”的择时筛选流程
为预测下周，你需要一个清晰的宏观、中观、微观的判断流程。

1.  宏观定仓位：综合全球风险偏好（VIX）、中美流动性数据，先决定下周整体是偏进攻（高仓位）还是防守（低仓位）。
2.  中观定方向：利用行业相对强度、资金流向、政策新闻，从60多个行业中筛选出3-5个处于上升趋势或刚刚突破的强势行业。用拥挤度数据排除过热赛道。
3.  微观定个股：在选定行业中，利用个股新闻（超预期财报、新产品） 作为催化剂，结合量价关系（放量突破形态）和北向/主力资金流入，最终锁定2-3只个股。
