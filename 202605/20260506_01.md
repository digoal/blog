## 背景  
  
  
最近在录制思维筑基系列, 因为只录了音频, 所以打算用多张图片轮播+音频的方式转换成视频, 再发视频号.  
  
本文分享一下如何"用多张图片轮播+音频的方式转换成视频"?  
  
环境, macmini m2 芯片, 可硬件加速转换过程.  
  
1、卸载已有的 homebrew 安装的 ffmpeg  
```  
brew uninstall --ignore-dependencies ffmpeg --force  
brew untap homebrew/core  
sudo rm -rf /usr/local/bin/ffmpeg /usr/local/bin/ffplay /usr/local/bin/ffprobe  
sudo rm -rf /opt/homebrew/bin/ffmpeg /opt/homebrew/bin/ffplay /opt/homebrew/bin/ffprobe  
```  
  
2、从 osxexperts 下载最新 ffmpeg, 选中苹果芯片版本.  
  
https://www.osxexperts.net/  
  
例如  
  
https://www.osxexperts.net/ffmpeg81arm.zip  
  
3、使用下载好的 ffmpeg  
```  
unzip ffmpeg81arm.zip  
sudo cp ~/Downloads/ffmpeg /usr/local/bin/ffmpeg  
sudo chmod +x /usr/local/bin/ffmpeg  
```  
  
4、配置隐私与安全性, 允许打开并执行刚刚下载的 ffmpeg  
  
**假设适配竖屏观看体验. 首先确保都是使用竖屏拍的照片, 压缩时也是使用横(小)竖(大)的分辨率比例.**  
  
**如果你要横屏观看体验, 反过来即可. 使用横屏拍的照片, 压缩时使用横(大)竖(小)的分辨率比例.**  
  
5、单图例子  
  
照片: 1.jpg  
处理后照片: 1.png  
音频: 1.m4a  
合成视频: 1.mp4  
  
5\.1、 压缩图片  
  
通常手机拍摄的照片为 jpg 格式.  
  
先将 jpg 转换成 png, 否则无法加速视频合成, 合成过程会很慢  
```  
ffmpeg -i 1.jpg 1.png  
```  
  
如果你还想继续加速合成过程, 可以对图片进行压缩转换, 把分辨率降低到 iphone 13 pro 的屏幕大小(`1170*2532`).  
```  
ffmpeg -i 1.jpg -vf "scale=1170:2532:force_original_aspect_ratio=decrease,pad=1170:2532:(ow-iw)/2:(oh-ih)/2,format=p010le" 1.png  
```  
  
5\.2、 合成视频  
  
正常语速  
```  
ffmpeg -hwaccel videotoolbox -loop 1 -i 1.png -i 1.m4a -shortest -c:v hevc_videotoolbox -b:v 2M -tag:v hvc1 -af loudnorm -c:a aac 1.mp4  
```  
  
1.6 倍语速  
```  
ffmpeg -hwaccel videotoolbox -loop 1 -i 1.png -i 1.m4a -shortest -c:v hevc_videotoolbox -b:v 2M -tag:v hvc1 -af "atempo=1.6, loudnorm" -c:a aac 1.mp4  
```  
  
6、多照片循环播放例子  
  
照片: 1.jpg 2.jpg 3.jpg  
处理后照片: 1.png 2.png 3.png  
音频: 1.m4a  
合成视频: 1.mp4  
  
一次转换多张照片  
```  
for i in {1..3}; do ffmpeg -i "$i.jpg" -vf "scale=1170:2532:force_original_aspect_ratio=decrease,pad=1170:2532:(ow-iw)/2:(oh-ih)/2,format=p010le" "$i.png"; done  
```  
  
图片必须按顺序命名为 1.png, 2.png, 3.png …，`FFmpeg` 会自动按数字顺序读取。`-framerate 1/5` 表示每秒 `0.2` 帧，即每张图停留 `5` 秒, `1/3`表示没张图片停留3秒。  
```  
ffmpeg -hwaccel videotoolbox -stream_loop -1 -framerate 1/3 -i %d.png -i 1.m4a -shortest -c:v hevc_videotoolbox -b:v 2M -tag:v hvc1 -af "atempo=1.6, loudnorm" -c:a aac 1.mp4  
```  
  
7、发布微信视频号, 似乎支持自动配字幕, 所以以上就不配字幕了.

逐条、逐项详细解释命令的每一个选项。  
  
  
## 📷 第一条命令：图片预处理  
  
```bash  
ffmpeg -i a.jpg -vf "scale=1920:1080:force_original_aspect_ratio=decrease,pad=1920:1080:(ow-iw)/2:(oh-ih)/2,format=p010le" a_ready.png  
```  
  
| 参数 | 含义 |  
|---|---|  
| `ffmpeg` | 调用 FFmpeg 程序 |  
| `-i a.jpg` | **i**nput，指定输入文件为 `a.jpg` |  
| `-vf "..."` | **v**ideo **f**ilter，对视频（此处是图片）应用滤镜链。滤镜之间用逗号 `,` 连接，从左到右依次执行 |  
| `scale=1920:1080` | 将图片缩放到 1920×1080 的框内 |  
| `force_original_aspect_ratio=decrease` | 缩放时**保持原图宽高比**。`decrease` 的意思是：如果原图和 1920×1080 的比例不一致，就**缩小到能完全装进这个框**，绝不裁剪或拉伸 |  
| `pad=1920:1080` | 把图片放在一个 1920×1080 的画布上 |  
| `(ow-iw)/2` | 水平居中：画布宽度 `ow` 减去图片宽度 `iw`，除以 2，就是左右留黑的宽度 |  
| `(oh-ih)/2` | 垂直居中：画布高度 `oh` 减去图片高度 `ih`，除以 2，就是上下留黑的高度 |  
| `format=p010le` | 将像素格式转换为 **10-bit YUV 4:2:0**，这是苹果 M2 的 HEVC 硬件编码器最喜欢的格式，后续编码可以零损耗直接处理 |  
| `a_ready.png` | 输出文件名。`.png` 格式可以无损保存 10-bit 色彩 |  
  
**这条命令的效果：**  
把一张任意分辨率的图片，变成一张 1920×1080 的 PNG，原图内容居中、比例不变、上下或左右自动加黑边，格式已为硬件加速编码做好准备。  
  
  
  
## 🎬 第二条命令：生成视频  
  
```bash  
ffmpeg -hwaccel videotoolbox -loop 1 -i a_ready.png -i a.m4a -shortest -c:v hevc_videotoolbox -b:v 2M -tag:v hvc1 -af "atempo=1.6, loudnorm" -c:a aac a.mp4  
```  
  
### 全局选项  
  
| 参数 | 含义 |  
|---|---|  
| `-hwaccel videotoolbox` | **h**ard**w**are **accel**eration，启用苹果的 VideoToolbox 硬件加速框架。让解码、编码尽可能走 M2 芯片的专用媒体引擎 |  
  
### 输入选项  
  
| 参数 | 含义 |  
|---|---|  
| `-loop 1` | 让单张图片**无限循环**。不加这个的话，图片只会显示一瞬间就没了 |  
| `-i a_ready.png` | 第一个输入文件：预处理好的图片 |  
| `-i a.m4a` | 第二个输入文件：音频 |  
| `-shortest` | 输出文件的时长 = **最短的那个输入**的时长。图片是无限循环的，所以视频长度 = 音频长度。音频播完，视频就结束 |  
  
### 视频编码选项  
  
| 参数 | 含义 |  
|---|---|  
| `-c:v hevc_videotoolbox` | **c**odec **v**ideo，视频编码器。使用苹果 M2 的**硬件 H.265/HEVC 编码器**。这是速度最快的选项 |  
| `-b:v 2M` | **b**itrate **v**ideo，视频码率。`2M` = 2000 kbps。对于 1080p 静态图片视频，这个码率画质已经非常清晰 |  
| `-tag:v hvc1` | 视频编码标签。`hvc1` 告诉播放器"这是标准的 HEVC"。确保在 iPhone / iPad / Mac 上直接播放，不会被识别为流媒体格式 |  
  
### 音频选项  
  
| 参数 | 含义 |  
|---|---|  
| `-af "atempo=1.6, loudnorm"` | **a**udio **f**ilter，音频滤镜链 |  
| `atempo=1.6` | **a**udio **tempo**，音频速度调整为**原来的 1.6 倍**。范围 0.5~2.0。自动保持音调不变，人声不会变尖 |  
| `loudnorm` | **loud**ness **norm**alization，响度标准化。分析整个音频，按 EBU R128 广播标准统一响度，让声音既饱满又不爆音 |  
| `-c:a aac` | **c**odec **a**udio，音频编码器。`aac` 是通用性最好的有损音频格式，所有设备都支持 |  
  
### 输出  
  
| 参数 | 含义 |  
|---|---|  
| `a.mp4` | 最终输出的视频文件名 |  
  
  
  
### 📊 整体流程总结  
  
```  
a.jpg (4032×3024 原始照片)  
    │  
    ▼  [scale → pad → format]  
a_ready.png (1920×1080, p010le, 居中+黑边)  
    │  
    ├─ 无限循环 ─┐  
    │            ├─→ [hevc_videotoolbox 硬件编码] ─→ a.mp4  
    │            │      视频: 1080p, HEVC, 2Mbps  
    a.m4a ───────┘  
    │              音频: 1.6倍速, 响度标准化, AAC  
    ▼  
  [atempo → loudnorm]  
```

## 第三条命令：多图片合成  
  
```bash  
ffmpeg -hwaccel videotoolbox -stream_loop -1 -framerate 1/5 -i %d.png -i 1.m4a -shortest -c:v hevc_videotoolbox -b:v 2M -tag:v hvc1 -af "atempo=1.6, loudnorm" -c:a aac 1.mp4  
```  
  
  
  
### 全局选项  
  
| 参数 | 含义 |  
|---|---|  
| `ffmpeg` | 调用 FFmpeg 程序 |  
| `-hwaccel videotoolbox` | **h**ard**w**are **accel**eration。启用苹果的 VideoToolbox 硬件加速框架，让解码和编码尽可能走 M2 芯片的专用媒体引擎，速度最快、功耗最低 |  
  
  
  
### 输入选项  
  
#### 第一个输入：图片序列  
  
| 参数 | 含义 |  
|---|---|  
| `-stream_loop -1` | 让这个输入流**无限循环**。`-1` 表示循环无数次。图片序列播完后自动从头再来，不会停止 |  
| `-framerate 1/5` | 设置帧率为 **0.2 fps**，即每秒播放 0.2 帧，等价于**每张图片停留 5 秒** |  
| `-i %d.png` | **i**nput，指定输入文件。`%d` 是一个通配符，FFmpeg 会按数字顺序匹配文件，比如 `1.png`, `2.png`, `3.png`... 并识别为一个完整的图片序列 |  
  
**这两者配合的效果：**  
```  
图片序列: [1.png 5秒] → [2.png 5秒] → [3.png 5秒] → 播完 → 从头循环  
              ↓              ↓              ↓  
          无限循环, 直到被 -shortest 终止  
```  
  
#### 第二个输入：音频  
  
| 参数 | 含义 |  
|---|---|  
| `-i 1.m4a` | 第二个输入文件：你的音频文件（语音备忘录录音）。它会作为最终视频的音轨 |  
  
  
  
### 时长控制  
  
| 参数 | 含义 |  
|---|---|  
| `-shortest` | 输出文件的时长 = **所有输入中最短的那个**。图片序列是无限循环的（时长无穷大），音频有固定时长，所以最终视频长度 = 音频长度。音频播完，视频立即结束，不会多出一帧 |  
  
  
  
### 视频编码选项  
  
| 参数 | 含义 |  
|---|---|  
| `-c:v hevc_videotoolbox` | **c**odec **v**ideo，指定视频编码器。使用苹果 M2 芯片的**硬件 H.265/HEVC 编码器**，编码速度极快，压缩率高 |  
| `-b:v 2M` | **b**itrate **v**ideo，视频目标码率。`2M` = 2000 kbps。对于静态图片轮播视频，这个码率已经能保证非常清晰的画质。图片越多、画面越复杂，可以适当提高（如 5M） |  
| `-tag:v hvc1` | 视频编码标签。`hvc1` 告诉播放器"这是标准的 HEVC 视频"。这个标签确保生成的视频在所有苹果设备（iPhone / iPad / Mac）上都能直接播放，不会因为缺少标签而被误识别 |  
  
  
  
### 音频滤镜选项  
  
| 参数 | 含义 |  
|---|---|  
| `-af "atempo=1.6, loudnorm"` | **a**udio **f**ilter，音频滤镜链。两个滤镜用逗号 `,` 连接，从左到右依次执行 |  
  
#### 滤镜 1：`atempo=1.6`  
  
| 子参数 | 含义 |  
|---|---|  
| `atempo` | **a**udio **tempo**，调整音频播放速度 |  
| `=1.6` | 速度倍数。`1.6` 表示加速到原来的 **1.6 倍**（比如原来 60 秒的录音，处理后变成 37.5 秒）。范围是 `0.5` ~ `2.0`。**关键特性：自动保持音调不变**，人声不会因为加速而变尖变细 |  
  
#### 滤镜 2：`loudnorm`  
  
| 子参数 | 含义 |  
|---|---|  
| `loudnorm` | **loud**ness **norm**alization，响度标准化。按照 EBU R128 广播标准，分析整个音频的响度，统一调整到标准水平 |  
  
**效果：**  
- 声音太小？自动放大。  
- 声音太大？自动衰减防止爆音。  
- 不同录音之间音量不一致？统一到相同的听感响度。  
  
  
  
### 音频编码选项  
  
| 参数 | 含义 |  
|---|---|  
| `-c:a aac` | **c**odec **a**udio，指定音频编码器。`aac`（Advanced Audio Coding）是通用性最好的有损音频格式，所有设备和平台都完美支持 |  
  
  
  
### 输出  
  
| 参数 | 含义 |  
|---|---|  
| `1.mp4` | 最终输出的视频文件名。格式为 MP4 容器，包含 HEVC 视频流 + AAC 音频流 |  
  
  
  
### 整体工作流程  
  
```  
[1.png, 2.png, 3.png...]          [1.m4a 8分56秒]  
        │                                │  
        ▼                                ▼  
  -stream_loop -1              atempo=1.6 (加速)  
  -framerate 1/5               loudnorm (响度标准化)  
  每张5秒，无限循环                    │  
        │                                │  
        └──────────┬─────────────────────┘  
                   ▼  
              -shortest  
          (取最短时长 = 音频长度)  
                   │  
                   ▼  
         hevc_videotoolbox 硬件编码视频  
         aac 编码音频  
                   │  
                   ▼  
              1.mp4 (最终视频)  
```  
  
  
  
### 常见参数组合速查  
  
| 需求 | 参数修改 |  
|---|---|  
| 每张图 3 秒 | `-framerate 1/3` |  
| 每张图 10 秒 | `-framerate 1/10` |  
| 2 倍速 | `atempo=2.0` |  
| 不需要倍速 | 去掉 `atempo=1.6,`，只保留 `-af loudnorm` |  
| 更高画质 | `-b:v 5M` |  
| 更小文件 | `-b:v 1M` |  
| 图片不循环，播完就停 | 去掉 `-stream_loop -1` |  
| 图片循环但只循环 3 次 | `-stream_loop 2`（0 是第一次，2 是额外再循环 2 次） |  
  
  
  
这条命令是你目前的最优方案：硬件加速、图片自动循环、音频倍速+响度标准化、文件小画质好。
