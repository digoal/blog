# 可视化模板库(mermaid / ascii / svg)

> 这是 SKILL.md 步骤 6 的扩展。每份预测报告**必须包含至少 2 个图**,优先级:四维雷达 mermaid > 价格刻度 ascii > 概率分布 svg。
> 写报告前先 Read 本文件,挑选适合的模板,把占位符 `___` 替换成实际值。

## 一、四维评分雷达图(mermaid,推荐)

### 模板 A:radar 图(mermaid 11+ 支持)

```mermaid
radar
    title 四维分析雷达图
    "大盘环境(30%)" : ___
    "资金行为(30%)" : ___
    "技术面(25%)" : ___
    "消息面(15%)" : ___
```

> 把 `___` 替换成 -2 ~ +2 的分值。注意:并非所有 mermaid 渲染器都支持 radar,如不支持改用模板 B。

### 模板 B:替代方案 - bar chart

```mermaid
%%{init: {"theme":"base"}}%%
graph LR
    classDef positive fill:#52c41a,color:#fff,stroke:#389e0d
    classDef negative fill:#ff4d4f,color:#fff,stroke:#a8071a
    classDef neutral fill:#bfbfbf,color:#fff,stroke:#595959

    A[大盘环境<br/>权重30%<br/>得分: +1] -->|加权 +0.30| Z
    B[资金行为<br/>权重30%<br/>得分: +2] -->|加权 +0.60| Z
    C[技术面<br/>权重25%<br/>得分: 0]   -->|加权 0.00| Z
    D[消息面<br/>权重15%<br/>得分: +1] -->|加权 +0.15| Z
    Z((加权得分<br/>+1.05<br/>强看多))

    class A,B,D positive
    class C neutral
```

## 使用建议

1. **每份报告至少 2 个图**(雷达 + 价格尺是黄金组合)
2. **mermaid 优先**,因为 Github / Notion / 大多数 markdown 渲染器都支持
3. **mermaid 不渲染时,降级用 ASCII**(永远兼容)
4. **SVG 适合作为概率分布、视觉冲击的辅助**
5. **图必须配文字说明**,不能光给图不解释
6. **图里的数字必须和正文一致**(占位符 `___` 必须替换)
