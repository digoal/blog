# Investor News Impact

## Overview

抓取最近 3 天内的新闻, 按主题分类总结, 用因果传导链推演对行业和上市公司的影响, 输出带图 markdown 日报到当前项目 `markdown/` 目录。

## Input

- 用户调用时可能传入 1 个行业名 (中文/英文皆可), 例如 `/investor-news-impact 半导体`
- 不传则使用 8 个默认主题 (见 `references/topics.md`)

## Workflow

### Step 1 · 抓取 (concurrent)

- 每个主题构造 1 个 `mcp__MiniMax__web_search` query: `<主题中文> <主题英文> 最近3天 新闻`
- 必须并发调用 (单条 message 内多条 tool_use), 不要串行
- 每个主题保留 8~15 条, 优先权威源 (Reuters / Bloomberg / WSJ / FT / 新华社 / 财新 / 36Kr / 证券时报 等)
- 用户传入具体行业时, 围绕该行业上下游和直接竞争者做 1~3 个 query 即可

### Step 2 · 分类与去重

- 按 8 大主题 (或用户行业) 二次分类
- 同一事件多源报道只保留信息最全的 1 条, 在摘要里注明「多源印证」
- 剔除超过 3 天的旧闻 (除非该事件仍在持续发酵)
- 时间窗冲突时, 以发布日期为准

### Step 3 · 逐条摘要

每条新闻输出一行, 格式:

```
- **<日期> <主题>** <一句话事件> — <数据/数字> — <来源>
```

### Step 4 · 全面总结

用一段连贯的散文 (300~500 字) 总结本周期的核心叙事主线, 包含:
- 跨主题共性 (例如「美联储转鸽 + AI 资本开支超预期」组合)
- 与上一周期 (若已有日报) 的连续性
- 未来 1~2 周需跟踪的催化剂

### Step 5 · 影响推演

对每个主题, 用 4 段式输出:

1. **传导链** — 新闻 → 中间变量 → 行业 → 上市公司, 至少 3 跳
2. **理论依据** — 引用经济学/金融学逻辑 (如「汇率 → 出口竞争力」「利率 → DCF 分母」)
3. **历史镜像** — 引用 1 个相似历史事件和数据佐证
4. **受益/承压标的** — 列出 A 股 / 港股 / 美股 3~5 个具体 ticker 或公司名

**图文规则**:
- 简单因果链用 `mermaid` flowchart / graph 内嵌
- 复杂产业链拓扑、供需矩阵、多空博弈图落地为独立 `.svg` 文件, 引用语法:
  `<img src="diagrams/<filename>.svg" alt="..." width="100%">`
- svg 文件统一放 `markdown/diagrams/`, 文件名用英文 kebab-case

### Step 6 · 写文件

- 文件名: `markdown/<YYYY-MM-DD>_投资者新闻影响日报.md`
- 当日无新闻也要写文件, 注明「过去 3 天无重大新闻」并给出建议跟踪清单
- 在文件开头列出全部参考链接和 svg 资产清单

## Resources

按需加载 references/, 不要一次全部塞入上下文:

- **`references/topics.md`** — 8 个默认主题的中英文搜索关键词、关键事件类型、可触发公司清单。Step 1 抓取前必读
- **`references/impact-framework.md`** — 影响推演的方法论、历史镜像案例库、上市公司映射表。Step 5 推演时按主题加载相关小节
- **`references/diagram-patterns.md`** — mermaid / svg 模板, 含 svg 文件命名、引用、渲染注意事项。Step 5 出图前加载

## Critical Rules

1. **必须用 `mcp__MiniMax__web_search`**, 禁止用内置 `WebSearch` (项目 CLAUDE.md 强制要求)
2. **金融数字必须核验**: IPO 状态、估值、市值、利率点等关键数据若拿不准, 主动二次搜索验证, 不要凭训练记忆下笔
3. **不编造公司/ticker**: 影响推演列具体上市公司时, 必须是公开可查的真实公司, 不确定就改用行业方向描述
4. **保持外科手术式克制**: 日报只覆盖用户指定或默认的主题, 不要自作主张扩展
5. **svg 外挂**: 任何 svg 图都要落地成独立文件再 `<img>` 引用, 禁止把整段 `<svg>` 塞进 markdown
