# 单个专家作答模板 (中间产物, 落盘保存)

> 每个专家严格按这个 6 节结构作答。这是步骤 3 的硬规范, 不可省略, 也不可调换顺序。
>
> **这份产出物会落盘保存** (见 SKILL.md 步骤 3.8), 不只是临时上下文:
>
> - **保存路径**: 当前项目的 `markdown/` 目录
> - **文件名**:
>   - 中文: `markdown/<主题>-专家<编号>-<角色>-<YYYYMMDD>.md`
>   - 英文: `markdown/<topic>-expert-<n>-<role>-<YYYYMMDD>.md`
> - **`<主题>`**: 与最终综合稿完全一致, 用 kebab-case
> - **`<编号>`**: 1, 2, 3..., 与综合稿中引入该专家的顺序一致
> - **`<角色>`**: kebab-case 的人设关键词, 例如 `逆向投资人`、`量化pm`、`hrd-p9`、`城市规划学者`
>
> **文件包含完整内容**: 1-6 节 (综合稿取材区) + 内部备注 + 第 7 节验证记录 (审计区)。两类内容都落盘, 但在文件内用清楚的标题层级隔开, 方便综合阶段只取 1-6 节, 又能保留完整审计线索。
>
> **不要在文件名带"草稿"二字** — 这就是定稿, 只是不直接呈现给用户。
