# 6 大场景报告模板

> 每个场景一个固定结构，便于复用和质量自检。

## 场景 1：投资（Investment）

### 结构

```markdown
## 投资：顺势而为的资产配置方向

### 1. 总判断
- 国家当前在 [周期/方向]，[时间窗] 内 [行业 A/B/C] 受益度最高
- 建议 [加配/减配/择时] + 关注 [板块/方向]

### 2. 增配方向
- 方向 1：[行业/板块] + 政策依据 + 历史回测（可选）
- 方向 2：...
- 方向 3：...

### 3. 减配/规避方向
- 方向 1：[行业/板块] + 退坡时间表
- 方向 2：...

### 4. 择时指标
- 政策节点：X 政策在 Y 月落地
- 数据指标：M2、社融、CPI、PPI 的 [阈值]
- 市场信号：板块估值 [百分位]、资金流向

### 5. 风险
- 政策周期风险
- 估值过高风险
- 流动性风险

### 6. 自查清单
- [ ] 投资期限 ≥ [时间]？
- [ ] 风险承受能力匹配？
- [ ] 不集中于单一行业？
- [ ] 留有现金应对回撤？
- [ ] 知道什么时候退出？
```

### 标的层级

- **板块/方向**（推荐）：半导体、AI、新能源、消费医疗、高端制造
- **ETF/指数**（可选）：不推荐具体代码
- **个股**（不推荐）：仅给行业方向

## 场景 2：创业方向（Entrepreneurship）

### 结构

```markdown
## 创业方向：顺势而为的赛道选择

### 1. 总判断
- 当前适合创业的赛道：[A/B/C]
- 不建议进入的赛道：[D/E]
- 灰度机会：[F]（需谨慎评估）

### 2. 推荐赛道
- 赛道 1：[名称] + 政策依据 + 市场规模 + 切入方式
- 赛道 2：...
- 赛道 3：...

### 3. 切入模式
- 模式 1：[B2B/B2C/SaaS/硬件/出海/...] + 优势
- 模式 2：...

### 4. 团队组建
- 核心能力 1：[技术/销售/运营/...]
- 核心能力 2：...
- 关键岗位：...

### 5. 资金策略
- 启动资金：[金额范围]
- 融资节奏：[天使/A轮 时间]
- 退出预期：[并购/IPO/...]

### 6. 风险
- 政策周期：政策可能转向
- 竞争格局：巨头/国资进入
- 能力匹配：团队经验不足

### 7. 自查清单
- [ ] 创始人有 5 年+ 相关经验？
- [ ] 至少有 6 个月生活费？
- [ ] 验证过最小可行产品（MVP）？
- [ ] 知道政策退坡时间？
- [ ] 有退出路径？
```

## 场景 3：商业思路（Business）

### 结构

```markdown
## 商业思路：存量生意的转型升级

### 1. 现有业务诊断
- 业务现状：[当前模式 + 收入/利润/客户]
- 政策匹配度：[高/中/低]

### 2. 政策窗口
- 国家鼓励：[方向] → 你能搭上车的切入点
- 国家限制：[方向] → 你需要转型的方向

### 3. 转型路径
- 路径 1（轻）：[现有业务 + 新增政策红利服务]
- 路径 2（中）：[现有业务 + 改造升级]
- 路径 3（重）：[新业务孵化]

### 4. 新业务方向
- 方向 1：[名称] + 切入点 + 资源需求
- 方向 2：...

### 5. 风险
- 现有业务被政策冲击
- 新业务投入过大
- 团队转型能力不足

### 6. 自查清单
- [ ] 现有业务现金流能撑 12 个月？
- [ ] 新业务有政策对接窗口？
- [ ] 团队愿意转型？
- [ ] 有试点客户？
- [ ] 知道什么时候止损？
```

## 场景 4：产品思路（Product）

### 结构

```markdown
## 产品思路：政策红利 + 用户痛点 = 商业机会

### 1. 用户痛点梳理
- 痛点 1：[人群/场景/未被满足的需求]
- 痛点 2：...
- 痛点 3：...

### 2. 政策红利对接
- 政策 1：[方向] + 时间窗 + 受益人群
- 政策 2：...

### 3. 产品机会
- 机会 1：[痛点 × 政策] → 产品形态 + 商业模式
- 机会 2：...

### 4. 商业模式
- 收入来源：[订阅/交易/广告/服务费/...]
- 客单价：[区间]
- 获客渠道：[渠道 1/2/3]

### 5. 合规风险
- 资质要求
- 数据合规
- 行业准入

### 6. 自查清单
- [ ] 痛点有具体人群验证？
- [ ] 政策红利有时间窗？
- [ ] 商业模式跑通？
- [ ] 合规要求清晰？
- [ ] 有壁垒？
```

## 场景 5：就业择业（Career）

### 结构

```markdown
## 就业择业：行业景气度 + 个人能力匹配

### 1. 行业景气度
- 上升行业：[A/B/C] + 增速 + 政策依据
- 平稳行业：[D/E] + 现状
- 退坡行业：[F/G] + 时间表

### 2. 岗位缺口
- 缺口最大的岗位：[技术/产品/销售/...]
- 具体岗位 1：[名称] + 薪资区间 + 技能要求
- 具体岗位 2：...

### 3. 城市选择
- 推荐城市：[城市 1/2/3] + 行业聚集 + 政策红利
- 不推荐：[城市 4/5] + 原因

### 4. 简历方向
- 强化方向 1：[项目/技能]
- 强化方向 2：...
- 弱化方向：[不相关经历]

### 5. 风险
- 入错行（朝阳行业中的夕阳岗位）
- 入错城（政策未落地的城市）
- 高估自身能力

### 6. 自查清单
- [ ] 行业在政策红利期？
- [ ] 岗位有真实缺口？
- [ ] 城市有产业聚集？
- [ ] 技能能迁移？
- [ ] 薪资匹配能力？
```

## 场景 6：学习规划（Study）

### 结构

```markdown
## 学习规划：政策红利 + 个人禀赋 = 长期赛道

### 1. 专业选择
- 推荐专业：[专业 1/2/3] + 政策依据 + 就业方向
- 谨慎专业：[专业 4/5] + 退坡原因
- 跨学科机会：[专业 A + B 的组合]

### 2. 院校选择
- 类型 1：研究型（985/211 + 双一流）
- 类型 2：应用型（行业特色院校）
- 类型 3：职业型（高职/技师）

### 3. 技能学习
- 硬技能 1：[编程/数据分析/...] + 学习路径
- 硬技能 2：...
- 软技能：[沟通/管理/...]

### 4. 证书/考证
- 高价值证书：[CPA/法考/...] + 政策红利
- 谨慎考证：[证书 X] + 退坡风险

### 5. 学历路径
- 本科 → 硕士 → 博士 的政策时间窗
- 在职/海外/跨专业的选择

### 6. 风险
- 选错专业（夕阳产业）
- 学历贬值（盲目追求高学历）
- 证书过剩（考完没用）

### 7. 自查清单
- [ ] 专业在政策红利期？
- [ ] 院校有产业对接？
- [ ] 技能可迁移？
- [ ] 证书有真实价值？
- [ ] 长期路径清晰？
```

## 跨场景通用要素

每个场景的报告都必须包含：

1. **总判断**（1 段，3-5 句话）
2. **机会/方向清单**（3-5 条）
3. **执行路径**（具体动作 + 时间表）
4. **关键指标**（领先指标，3-5 个）
5. **风险**（3 类）
6. **自查清单**（5 条 yes/no）
