# 势—道—术 推演框架

> 5 步法把"国家想干什么"翻译成"个人能干什么"。

## 总体框架

```
[势] 国家战略方向     ← 5-10 年
       ↓
[道] 产业链结构变化   ← 3-5 年
       ↓
[术] 个人/组织动作    ← 1-3 年
       ↓
[推] 因果链 + 风险标注
       ↓
[案] 历史案例对照
```

## Step 1：定位势（5-10 年时间窗）

**问题**：当前国家处于什么周期？方向是什么？

### 1.1 周期判断

把当前五年规划定位到产业生命周期的 4 个阶段：

| 阶段 | 特征 | 政策动作 | 个人策略 |
|---|---|---|---|
| **导入期** | 投入大、产出小 | 国家强力补贴 | 早期布局，吃红利 |
| **成长期** | 增长快、竞争激烈 | 国家规范+扶持头部 | 跟头部，谨慎选股 |
| **成熟期** | 增速放缓、利润集中 | 国家维持+防内卷 | 选龙头，看分红 |
| **退坡期** | 补贴退坡、产能过剩 | 国家引导转型 | 退出/转型/出海 |

### 1.2 方向判断

从五年规划 + 历次全会公报提取 3-5 个**核心方向**：
- 用 1 个动词概括每个方向（"突破"、"升级"、"扩容"、"替代"、"出海"）
- 标注时间表（哪一年实现什么目标）
- 标注定量指标（GDP 占比、研发投入、产能、出口）

### 1.3 输出模板

```markdown
### 势：国家当前在 [周期阶段]，核心方向是 [方向 1/2/3]
- 方向 1：[动词] + [行业] + 目标年 + 关键指标
- 方向 2：[动词] + [行业] + 目标年 + 关键指标
- 方向 3：[动词] + [行业] + 目标年 + 关键指标
- 关键文件：[文件 1] [日期]、[文件 2] [日期]...
```

## Step 2：识别道（3-5 年时间窗）

**问题**：产业链的钱/政策/人/技术如何重塑？谁受益、谁受损？

### 2.1 4 维分析

| 维度 | 看什么 | 抓取信号 |
|---|---|---|
| **钱** | 财政/产业基金/信贷往哪流 | 财政信号 + 货币信号 |
| **政策** | 鼓励/限制/淘汰/试点 | 产业信号 + 区域信号 |
| **人** | 人才/教育/就业往哪流 | 民生信号 + 科技信号 |
| **技术** | 卡脖子/攻关/转化 | 科技信号 + 对外信号 |

### 2.2 受益/受损矩阵

把每个行业按"受益度"打分（-3 到 +3）：
- +3：明确鼓励 + 大量资金 + 政策强力
- +2：鼓励 + 资金支持
- +1：默许/间接利好
- 0：中性
- -1：限制
- -2：淘汰
- -3：明确淘汰 + 时间表

### 2.3 产业链图谱

用 Mermaid `flowchart` 画出产业链：
- 上游（原料/设备）
- 中游（生产/集成）
- 下游（应用/服务）
- 标注每个节点的受益度
- 标注钱/政策/技术的流向箭头

### 2.4 输出模板

```markdown
### 道：产业链如何重塑

**鼓励类行业**（受益度 ≥ +2）：
- [行业 A]：钱从 X 流向 Y，政策给 Z，技术突破 W → 受益环节 [上游/中游/下游]
- [行业 B]...

**限制/淘汰类行业**（受益度 ≤ -1）：
- [行业 C]：退坡时间表 X，过渡期 Y，转型方向 Z
- [行业 D]...

**中性/灰度类行业**：
- [行业 E]：边界 X，机会 Y，风险 Z

**产业链图谱**：
```mermaid
flowchart LR
    A[上游：原料] -->|资金 +3| B[中游：生产]
    B -->|政策 +3| C[下游：应用]
```
```

## Step 3：匹配术（1-3 年时间窗）

**问题**：6 大场景下，用户具体能做什么？

参考 [user-profile-mapping.md](user-profile-mapping.md) 把用户画像与机会匹配。

### 3.1 匹配度打分

| 用户条件 | 匹配机会 | 打分依据 |
|---|---|---|
| 行业经验 | 同行业的机会 | 经验直接迁移 +3 |
| 技能匹配 | 政策红利需要的技能 | 技能对口 +2 |
| 城市匹配 | 政策试点城市 | 城市对口 +2 |
| 资源匹配 | 资金/渠道/人脉 | 资源够用 +1 |
| 爱好契合 | 兴趣方向的机会 | 长期投入意愿 +1 |

### 3.2 6 大场景落地

详见 [scenario-templates.md](scenario-templates.md)

### 3.3 输出模板

```markdown
### 术：场景 X 的具体动作

**机会判断**：国家趋势给 [场景 X] 开了 [时间窗] 的窗口

**可执行动作**（1-3 个月）：
1. 动作 1：具体做什么
2. 动作 2：具体做什么
3. 动作 3：具体做什么

**关键指标**（领先指标）：
- 政策：X 政策落地节点
- 数据：X 行业增速
- 招聘：X 类岗位需求
- 订单：X 客户/政府订单

**风险与避坑**：
- 风险 1：政策周期风险
- 风险 2：执行落差风险
- 风险 3：能力不匹配风险
```

## Step 4：标注风险

每个机会必须标 3 类风险：

| 风险类型 | 含义 | 识别方法 |
|---|---|---|
| **政策周期风险** | 政策可能退坡/转向 | 看文件用语（"有序"、"试点"、"过渡期"） |
| **执行落差风险** | 政策与实际执行有差距 | 看历史政策落地情况 |
| **能力不匹配风险** | 用户/组织能力跟不上 | 自查技能/资源/经验 |

## Step 5：类比历史

从 [historical-case-bank.md](historical-case-bank.md) 找 1-3 个相似历史案例：

- 上次五年规划时的类似政策
- 相似国家（美/日/韩/欧）的类似阶段
- 相似行业的过去周期

每个案例给：
- 时间：什么时候发生
- 政策：当时怎么做
- 结果：实际效果
- 启示：对我们这轮的启示

## 因果链自检

写完推演后用这个清单自查：

- [ ] 势 → 道：国家方向有具体数据/文件支撑？
- [ ] 道 → 术：产业链变化有具体环节落地？
- [ ] 术 → 用户：动作有具体可执行步骤？
- [ ] 风险：3 类风险是否标注？
- [ ] 历史：至少 1 个对照案例？
- [ ] 闭环：从"国家说 A"到"用户做 D"是否有完整因果链？
